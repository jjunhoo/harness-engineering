# Incident to Rule

## 입력

- 사건 요약(비밀 없이)
- 영향(가용성·데이터·보안)
- 근본 원인 가설

## 출력 (하나 이상)

- `rules/`에 새 제약 또는 기존 문서 개정
- `evaluations/`에 회귀 테스트·체크리스트·시나리오
- 필요 시 `architecture/adr/`에 결정 기록

## 완료 정의

동일 실패를 **CI 또는 체크리스트**로 재현·차단할 수 있다.
