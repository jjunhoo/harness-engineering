# 레이어드 모델

의존성은 아래 **위에서 아래로만** 허용한다. 역방향 금지.

```
UI
 ↑
Runtime (HTTP 서버, 작업 큐 워커, CLI 진입점)
 ↑
Service (유스케이스, 트랜잭션 경계)
 ↑
Repo (DB·외부 API 접근, 매퍼)
 ↑
Config (환경·플래그, 검증된 설정 객체)
 ↑
Types (도메인 타입, 공용 DTO, 에러 타입)
```

## 레이어별 책임

| 레이어 | 허용 | 금지 |
|--------|------|------|
| Types | 순수 타입, 작은 유틸 | I/O, 프레임워크 임포트(필요 시 최소) |
| Config | 로드·검증 | 비즈니스 규칙 |
| Repo | 영속성·원격 호출 | UI 관심사 |
| Service | 오케스트레이션 | HTTP 세부 |
| Runtime | 어댑터 바인딩 | 비즈니스 규칙 누수 |
| UI | 표현·입력 | 직접 Repo 호출 |

## 로깅 표준

- 경계: `Service` 이상에서 **도메인 이벤트** 로깅 권장
- `Repo`는 **지연·에러 코드** 수준
- 공통 필드: `correlation_id`, `service`, `version` (`RELIABILITY.md` 정합)
