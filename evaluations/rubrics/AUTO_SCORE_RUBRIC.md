# 자동 평가 루브릭 — 코드 품질 · 구조 · 성능

`evaluations/scripts/auto-score.sh`가 산출하는 **0–100 점수**와 `QUALITY_SCORE.md`의 정성 차원을 연결한다.

## 차원 정의

| 차원 | 영문 키 | 의미(본 저장소 기준) | 주요 신호 |
|------|---------|----------------------|-----------|
| 구조 | `structure` | 하네스 문서·디렉터리·스킬·CI 게이트 정합 | `harness-validate.sh` 통과 여부 |
| 코드 품질 | `code_quality` | 셸 스크립트 정적 분석·비밀 패턴 | `shellcheck`(선택), 비밀형 문자열 스캔 |
| 성능 | `performance` | 평가 파이프라인 자체의 지연(피드백 속도 프록시) | `auto-score` 측정 구간 경과 시간 |

## 점수 해석

- **100**: 해당 차원에서 자동 검사가 모두 통과(또는 측정 상한 내).
- **55 미만(가중 총점)**: `config/auto-score.json`의 `gates.min_overall` 미달 시 CI 실패(설정 가능).
- **구조 0**: 하드 게이트 실패와 동일 — 즉시 수정 대상.

## 코드 품질 세부

1. **shellcheck**가 설치되어 있으면 `scripts/**/*.sh`, `evaluations/scripts/**/*.sh`에 대해 실행. 파일당 실패 시 가중 감점(`config` 참고).
2. 설치되어 있지 않으면 고정 감점(문서화된 기본값) — 로컬 개발 환경에서 설치 권장.
3. **비밀 패턴**: 셸 스크립트에서 개인키·AWS 키·슬랙 토큰 유사 패턴이 발견되면 해당 차원 **0점** 및 `gates.hard_secret_fail`.

## 성능 세부

- 측정 구간: 구조 검증 + 코드 품질 검사 + 점수 집계까지의 **벽시계 시간**.
- 임계: `excellent_ms` 이하 ≈ 만점 근접, `acceptable_ms` 초과 시 급감(세부는 스크립트 내 피스와이즈 함수).

## 제품 코드 확장

`src/`, `go.mod`, `package.json` 등이 생기면 `evaluations/scripts/plugins/`에 언어별 점검을 추가하고 `auto-score.sh`에서 호출하도록 확장한다. 본 루브릭은 **메타 하네스 저장소**에 맞춘 기본값이다.
